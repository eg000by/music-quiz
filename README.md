# 🎵 Egorii — музыкальная викторина

> Угадай трек по короткому отрывку и год его выпуска. Один, с друзьями или со случайным соперником.

**[egorii.fun](https://egorii.fun)** — быстрая музыкальная викторина в реальном времени.
Слушаешь короткий отрывок песни, угадываешь название и год выпуска — соло, с друзьями по
коду лобби или против случайного соперника. Веб-приложение полностью бесплатное и работает
**без собственного бэкенда**, на бесплатном тарифе Firebase (Spark).

Интерфейс **двуязычный** (русский / английский, язык определяется автоматически).

## ✨ Возможности

- **Мультиплеер-лобби** — создай лобби, поделись 4-значным кодом и играй в реальном
  времени (2–4 игрока). Живая «пирамида лидерства» показывает, кто впереди прямо по ходу матча.
- **Быстрый матч** — клиентский матчмейкинг сводит тебя со случайным соперником для
  короткой дуэли (без сервера).
- **«Трек дня»** — один общий трек на весь день в стиле Heardle: 5 попыток, каждая
  открывает отрывок подлиннее, есть серия побед (streak) и эмодзи-сетка для шеринга результата.
- **Угадай год** — второй шаг раунда: ставишь год выпуска на слайдере; при показе ответа
  шкала «приближается» к догадкам игроков.
- **Режим «Эволюция трека»** — звук стартует приглушённым и проясняется к концу раунда
  (Web Audio API).
- **Музыкальные паки** — 16 подборок (мировые хиты, рок-классика, русский поп/рок, K-pop,
  аниме-опенинги, саундтреки к фильмам, 90-е/2000-е/2010-е…). Легко расширяются.
- **Профили и никнеймы** — вход через Google, чтобы задать ник, сохранять счёт в
  таблице лидеров и держать серию «Трека дня» на всех устройствах.
- **Лёгкая система друзей** — недавние соперники и приглашения «сыграть снова» в один клик
  с realtime-уведомлениями в приложении.
- **Таблица лидеров**, **политика конфиденциальности с баннером согласия**, кнопка **доната**.

## 🛠️ Технологии

- **Фронтенд:** React 19, Vite 8 (Rolldown), react-router-dom 7, чистый JavaScript.
  Без UI-фреймворка; своя лёгкая i18n и набор иконок.
- **Бэкенд:** Firebase — Auth (анонимный + Google), Firestore (состояние игры в реальном
  времени), Hosting. **Только тариф Spark** (без Cloud Functions / Storage).
- **Аудио и обложки:** бесплатный [iTunes Search API](https://performance-partners.apple.com/search-api)
  (30-секундные превью через JSONP — без ключей и прокси).
- **Безопасность:** App Check (reCAPTCHA v3), усиленные Firestore Security Rules, строгий
  CSP и security-заголовки, аналитика включается только после согласия на cookie.
- **Производительность:** code-splitting, self-hosted шрифты, параллельное разрешение треков.

## 🚀 Локальный запуск

```bash
cp .env.example .env      # впиши свой Firebase web config (+ опц. App Check / GA / URL доната)
npm install
npm run dev               # http://localhost:5173
```

Создай проект в Firebase (хватит тарифа Spark), включи **Authentication → Google + Anonymous**
и **Firestore Database**, затем скопируй значения `firebaseConfig` веб-приложения в `.env`.
Чтобы проверить мультиплеер на одной машине, открой второе окно в приватном/инкогнито-профиле.

## 📦 Деплой (Firebase Hosting, бесплатно)

```bash
npm install -g firebase-tools     # один раз
firebase login
# укажи свой Project ID в .firebaserc
npm run build
firebase deploy                   # выкладывает dist/ + firestore.rules
```

После первого деплоя добавь домен `*.web.app` (и свой кастомный домен) в
**Authentication → Settings → Authorized domains**, чтобы заработал вход через Google.

## 🎶 Как добавить музыку

Редактируй [src/data/packs.js](src/data/packs.js). Каждая песня — это `{ title, artist }`:
`title` — это ответ, который видят игроки, `artist` используется только чтобы найти трек в
iTunes. Минимум 6 песен в паке; треки, не найденные в iTunes, тихо пропускаются. Каждому
паку задай `name` (RU) и `nameEn` (EN). Настройки дуэли/быстрого матча и число раундов —
в [src/services/lobby.js](src/services/lobby.js).

## 📝 Примечания

- Время синхронизируется с серверным временем Firestore, чтобы таймеры раунда у обоих
  игроков совпадали.
- Переходами между раундами управляет клиент-хост, поэтому его вкладка должна оставаться
  открытой в течение матча.
- Загрузка своих mp3 не используется: Firebase Storage требует платного тарифа Blaze —
  вместо этого превью берутся из iTunes.
